# Contributing to Police Cybercell Portal

Thank you for your interest in contributing to the Police Cybercell Portal! We welcome contributions to make this project better.

## How to Contribute

1.  **Fork the repository** to your own GitHub account.
2.  **Clone the project** to your local machine.
3.  **Create a new branch** for your feature or bug fix:
    ```bash
    git checkout -b feature/amazing-feature
    ```
4.  **Make your changes** and commit them with descriptive commit messages.
5.  **Push your changes** to your fork:
    ```bash
    git push origin feature/amazing-feature
    ```
6.  **Open a Pull Request** (PR) to the main repository.

## Development Guidelines

### Code Style
- We use **ESLint** and **Prettier** to maintain code quality and formatting.
- Please ensure your code passes linting before submitting a PR:
    ```bash
    npm run lint
    ```

### Project Structure
- Keep components modular and reusable.
- Place new components in the `src/components` directory.
- Use meaningful variable and function names.

### Commit Messages
- Use clear and concise commit messages.
- Start with a verb (e.g., "Add", "Fix", "Update").

## Reporting Issues
If you find a bug or have a feature request, please open an issue in the repository with details about the problem or suggestion.
